# Study Guide: Web Infrastructure Fingerprinting & Banners (Phase 8)

This study guide explains the systems engineering, packet inspection, and defensive hardening configurations implemented in **Phase 8** of the Website Security Analyzer.

---

## 1. Active vs. Passive Fingerprinting

Identifying the technologies running a target website is a critical part of security audits. It helps security teams catalog their inventory and patch versions, and helps attackers find entry points.
* **Passive Fingerprinting:** Analyzing network traffic passively (e.g. inspecting standard TCP window parameters, IP TTL values, or capturing normal request/response packets flowing on the network). No queries are generated by the auditor.
* **Active Fingerprinting:** Actively sending requests to the host and analyzing the responses (e.g. requesting page assets, checking default cookies, sending malformed requests to trigger error pages, and reading headers).

---

## 2. Banner Grabbing & Info Disclosures

Web server daemons (like Apache, Nginx, Microsoft-IIS) and language engines (like PHP, ASP.NET) often disclose their presence and exact version numbers in HTTP responses.

```
+--------------------------------------------------------------------------------+
|  HTTP Response Headers                                                         |
|  ---------------------                                                         |
|  HTTP/1.1 200 OK                                                               |
|  Date: Tue, 02 Jun 2026 13:56:17 GMT                                           |
|  Server: nginx/1.24.0                     <-- Web server + exact version leak! |
|  X-Powered-By: PHP/7.4.3                  <-- Language engine + version leak!  |
+--------------------------------------------------------------------------------+
```

### The Security Risk: CVE Identification
The Common Vulnerabilities and Exposures (CVE) system is a catalog of publicly disclosed security flaws.
* If a target exposes `Server: nginx/1.24.0`, an attacker knows that version was released around 2023. They can query CVE databases for all vulnerabilities affecting Nginx 1.24.x (such as HTTP/3 Rapid Reset or memory corruption bugs) and execute tailored exploits directly.
* Exposing software names and versions is classified as **Information Disclosure**.

---

## 3. Session Cookie Audits

Web applications use cookies to track user sessions. Standard application servers set characteristic default cookie names in the `Set-Cookie` header:

* **`PHPSESSID`:** Indicates a PHP-based application layer (e.g., WordPress, Drupal, or raw PHP scripts).
* **`JSESSIONID`:** Indicates a Java Servlet container (e.g., Apache Tomcat, WildFly, or JBoss).
* **`ASP.NET_SessionId` / `ASPSESSIONID`:** Indicates a Microsoft ASP.NET backend running on IIS.
* **`session` / `session_id`:** Often indicates Python micro-frameworks (Flask, Django) or Node.js.

By auditing the names of active cookies, security analysts can identify the underlying language engine even if headers like `Server` are stripped.

---

## 4. HTML Source Scraping (Generator Meta Tags)

Many Content Management Systems (CMS) insert creator signatures directly into the HTML head section to track market share.

* **WordPress tag:** `<meta name="generator" content="WordPress 6.4.2" />`
* **Drupal tag:** `<meta name="generator" content="Drupal 9 (https://www.drupal.org)" />`

We parse the raw HTML code of the index template using regular expressions. If a generator meta tag is present, we extract the CMS name and version to report patch levels.

## 5. Host Hardening Configurations

Secure configurations require minimizing header information leakages (practicing defense-in-depth, not security through obscurity alone):

### A. Nginx Configuration
Disable version banners by adding this directive in the `http` block of `nginx.conf`:
```nginx
http {
    server_tokens off;  # Replaces 'nginx/1.24.0' with 'nginx'
}
```

### B. Apache Configuration
Disable banners by editing `httpd.conf` or `security.conf`:
```apache
ServerTokens Prod    # Replaces 'Apache/2.4.41 (Ubuntu)' with 'Apache'
ServerSignature Off  # Disables footer signatures on error templates
```

### C. PHP Configuration
Disable powered-by headers in `php.ini`:
```ini
expose_php = Off     # Completely strips the X-Powered-By header
```
